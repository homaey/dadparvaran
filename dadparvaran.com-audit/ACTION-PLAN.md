## فاز ۱ — بحرانی (هفته اول)

### ۱.۱ رفع ۸,۱۳۵ عنوان تکراری صفحات قانون — ✅ انجام شد (`b3e9dbf`)
**تأثیر: بسیار زیاد | تلاش: کم (~۱ ساعت)**

> عنوان، og:title، description و H1 حالا نام قانون را شامل می‌شوند. description از متن
> واقعی ماده ساخته می‌شود. تأیید روی production: ۴۰ صفحه از پرتداخل‌ترین برچسب‌ها
> (`ماده-1`, `ماده-2`, `ماده-5`, `ماده-10`) → ۴۰ عنوان یکتا، صفر تکرار.

فایل: `src/app/[locale]/(public)/laws/[lawSlug]/[articleSlug]/page.tsx` → `generateMetadata`

```
فعلی:  ماده ۶۲۳ | دادپروران مهر ایران
هدف:   ماده ۶۲۳ قانون مجازات اسلامی — متن و شرح | دادپروران
```

همزمان `<h1>` نیز باید نام قانون را شامل شود، و meta description از متن واقعی ماده (۱۲۰–۱۵۵ کاراکتر) ساخته شود نه الگوی ثابت.

با یک تغییر، ۹۱٪ مشکل عنوان تکراری سایت حل می‌شود.

### ۱.۳ تصمیم درباره ۸,۹۲۷ صفحه ماده قانون
**تأثیر: بسیار زیاد | تلاش: متوسط**

این صفحات نازک (میانه ۱۶۶ کلمه) و تکراری محتوای صفحه والد هستند. دو مسیر:

- **مسیر الف (سریع، محافظه‌کارانه):** `noindex, follow` روی صفحات تکی مواد + حذف از سایت‌مپ. صفحه قانون که متن کامل دارد رتبه می‌گیرد. بودجه کراول آزاد می‌شود.
- **مسیر ب (بلندمدت، ارزش بیشتر):** غنی‌سازی تدریجی مواد پرجستجو (مثلاً ۲۰۰ ماده پربازدید) با تفسیر، رویه قضایی، مثال کاربردی و پرسش‌های متداول — سپس ایندکس همان‌ها.

**توصیه:** ترکیب — الف را فوراً اجرا کنید، سپس ب را برای مواد منتخب.

> نیازمند تصمیم شما؛ این تغییر روی ایندکس‌شدن ~۹,۰۰۰ صفحه اثر می‌گذارد و برگشت آن زمان‌بر است.
